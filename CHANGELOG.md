# Changelog

## v3.11.2 - Version Integrity + Report Sync

### Fixed

- Fixed version mismatch between App shell and Daily Report body.
- Added single version source: `src/radar/version.py`.
- Invalidates stale `dashboard_data.json` when payload version differs from current app version.
- Prevents stale `daily_report.md` from being displayed when it was generated by an older release.
- Upgrade script now removes runtime output files after release installation.
- Added regression tests for version consistency across payload, CLI/report helpers, and generated Markdown.

### Notes

This is a trust hotfix. Users must never see mixed versions such as app `3.11.1` with report `3.9.0`.
