# Release v3.11.2 - Version Integrity + Report Sync

## Purpose

Fix a trust-breaking issue where the Streamlit app could show the latest app shell version while the embedded Daily Report still displayed an older version.

## Scope

- Single version source.
- Report version sync.
- Runtime output invalidation.
- Upgrade-time output cleanup.
- Regression tests.

## Acceptance Criteria

- App title and Daily Report title always display the same version.
- Old `output/dashboard_data.json` is ignored if version mismatch exists.
- Old `output/daily_report.md` is not displayed if generated by an older version.
- CLI produces current-version payload and report.
